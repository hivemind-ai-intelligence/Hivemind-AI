## ▸ Overview

**Hivemind AI** is a full-stack, type-safe AI product platform built as a single pnpm monorepo. It pairs a modern React front end with an Express API, an OpenAI-backed AI layer, and a PostgreSQL database managed through Drizzle ORM &mdash; with the API contract generated from a single OpenAPI specification so the client and server never drift apart.

The public site presents the Hivemind AI product: an overview of the company, its services, pricing, and roadmap, alongside an interactive AI chat experience and an admin dashboard for managing site content.

## ▸ Why Hivemind AI

- **▸ One contract, zero drift** &mdash; the OpenAPI spec is the single source of truth; client hooks and server types are both generated from it.
- **▸ Type safety from database to UI** &mdash; Drizzle, Drizzle-Zod, and Zod share the same schemas across every layer of the stack.
- **▸ Composable by design** &mdash; the monorepo isolates the web app, API server, and shared libraries so each can evolve independently.
- **▸ Built for a real product, not a demo** &mdash; structured logging, health checks, and a production build pipeline are part of the foundation, not an afterthought.

## ▸ Architecture Highlights

- **◆ Monorepo workspace** &mdash; `artifacts/` holds deployable services (the web app and the API server); `lib/` holds shared, framework-agnostic packages consumed by them.
- **◆ Generated API client** &mdash; `@workspace/api-spec` defines the OpenAPI contract; `@workspace/api-client-react` and `@workspace/api-zod` are generated from it, so the frontend never hand-writes request types.
- **◆ Shared database package** &mdash; `@workspace/db` centralizes the Drizzle schema and connection so both the API server and tooling scripts use one definition of the data model.
- **◆ Independent build &amp; run per service** &mdash; the web app builds with Vite, the API server bundles with esbuild, and each runs on its own port behind Replit's routing.

```
├── scripts/                 # Workspace-level scripts
├── pnpm-workspace.yaml       # pnpm workspace & dependency catalog
├── package.json              # Root workspace scripts (build, typecheck)
└── replit.md                  # Project operating notes
```

## ▸ Getting Started

### Prerequisites

- Node.js 24+
- pnpm
- A PostgreSQL database (connection string via `DATABASE_URL`)

### Installation

```bash
pnpm install
```

<br/>

## ▸ Local Development

Run the API server:

```bash
pnpm --filter @workspace/api-server run dev
```

Run the web app:

```bash
pnpm --filter @workspace/hivemind run dev
```

Push the database schema (development only):

```bash
pnpm --filter @workspace/db run push
```

Regenerate the API client from the OpenAPI spec after changing it:

```bash
pnpm --filter @workspace/api-spec run codegen
```

Type-check the full workspace:

```bash
pnpm run typecheck
```

<br/>

## ▸ Deployment

Hivemind AI is deployed and running in production. The web app and API server are built independently &mdash; the web app compiles to a static bundle via Vite, and the API server bundles to a single ESM entrypoint via esbuild &mdash; so each service can be deployed and scaled on its own.
